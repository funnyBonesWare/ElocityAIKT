Proof: Jira → Cursor → GitLab → Tests → Ship

This is the exact workflow we run every day. Each stage produces visible, trackable output. Click any screenshot to see full details.

Stage 1: Jira — Requirement & Planning

What Happens Here
Input: CTO or Product Manager creates a Jira ticket with clear requirements, acceptance criteria, and priority.
Detail: Ticket includes user story, expected behavior, edge cases, and links to Confluence design docs.
Output: A fully scoped task ready for an engineer to pick up and start building.

Stage 2: Cursor — AI-Powered Development

What Happens Here
Input: Engineer opens Cursor, describes the task to the AI agent with Jira context.
AI does: Reads the full codebase, understands patterns, generates React components, API integrations, database queries, and business logic across multiple files.
Human does: Reviews AI output, adds domain knowledge, fixes edge cases, optimizes. ~20-30% human refinement.
Output: Production-ready code — pushed to a GitLab branch.

Stage 3: GitLab — AI Review & Merge Request

What Happens Here
Input: Code pushed to GitLab triggers an MR with AI-powered review.
AI does: Scans every changed line — checks for security issues, performance problems, code quality, and best practice violations. Generates a review summary.
Human does: Senior engineer reads the AI summary, verifies flagged items, checks business logic, and approves or requests changes.
Output: Approved MR ready for testing — with a full AI review trail.

Stage 4: Playwright MCP & Maestro — Test Evidence

What Happens Here
Input: Approved MR triggers automated E2E tests.
Playwright MCP: Opens a real browser, navigates pages, fills forms, clicks buttons, asserts expected behavior. Captures screenshots at every step as proof.
Maestro: Runs the same flow on mobile (iOS + Android) — app launch, gestures, navigation, data display.
If tests fail: AI agent reads the failure, fixes the code, and re-runs — automatically. Loop continues until all pass.
Output: Complete evidence package — screenshots, pass/fail, timings — attached to the MR.

Stage 5: Human Verify & Ship

What Happens Here
Input: MR with passing AI review + passing E2E tests + screenshot evidence.
Human does: Tech lead reviews the evidence package — confirms screenshots look correct, test coverage is adequate, business logic is sound.
Decision: Approve → merge → auto-deploy to staging → final QA → production.
Principle: AI does 90% of the work. Humans make the final call. Nothing ships without a human "yes."
