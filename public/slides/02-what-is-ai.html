What is AI? — In Simple Terms

Four building blocks power every "AI-powered" tool today. Here's what each one actually means, with real examples.

LLMs (Large Language Models)
In plain English: A program trained on billions of pages of text. It can read, understand, and write human language — like a very fast colleague who has read every document ever written.

Real tools: ChatGPT, GitHub Copilot, Claude, Gemini. Companies use these daily for drafting emails, summarizing docs, generating code, and answering questions from internal knowledge bases.
AI Agents
In plain English: An LLM that can take actions — not just talk. It thinks, acts, checks, and loops until a task is done. It can browse files, run commands, call APIs, and make multi-step decisions autonomously.

Real tools: Cursor (our IDE) is an AI agent — it reads our codebase, plans changes, writes code, and runs tests. Other examples: Devin, support bots that actually resolve tickets.
MCP (Model Context Protocol)
In plain English: A universal adapter that lets AI talk to your real tools — Jira, GitLab, Confluence, databases — so it can read and act on live data instead of guessing from copy-pasted text.

How we use it: Our Cursor agent uses MCP to directly read Jira tickets, pull Confluence docs, and push code to GitLab — all without manual copy-paste. One prompt triggers the full chain.
Evidence Loops
In plain English: After AI generates something, automated checks prove it works — not just trust it. Then a human verifies before it ships. The output is evidence, not hope.

How we use it: We use Playwright MCP — the AI agent triggers E2E tests directly via MCP. Playwright runs the app, checks behavior, captures screenshots. Then an engineer reviews the evidence and gives final sign-off. No code ships without both automated proof and human approval.

Real Example: "Build a Charger Analytics Dashboard"

Here's how all 4 building blocks work together when a developer picks up a real Jira ticket.

LLM
Read the Jira ticket and existing context to understand what to build and generate an implementation plan.
Agent
Cursor wrote React components, API calls, chart config, and filter logic across multiple files autonomously.
Evidence
Playwright MCP ran E2E tests automatically. Dashboard verified: loads, filters, charts, data. Human engineer reviewed evidence and gave final sign-off before merge.
